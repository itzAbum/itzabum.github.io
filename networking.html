Networking Concepts

DNS Resolution

This website is hosted on GitHub Pages.When a user enters itzabum.github.io, DNS resolves the domain to GitHub Pages’ IP addresses. Running nslookup itzabum.github.io returned multiple IPv4 and IPv6 addresses, showing that GitHub uses a global CDN.

Command used:

nslookup itzabum.github.io

IP Addressing

The DNS lookup returns an IPv4 and/or IPv6 address. This shows where the website is hosted on the internet.

From the screenshot above we can see:

IPv4: 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153

IPv6: 2606:50c0:8000::153, 2606:50c0:8001::153, 2606:50c0:8002::153, 2606:50c0:8003::153

GitHub Pages uses multiple addresses for redundancy and load balancing.

Protocols (HTTP/HTTPS)

This website uses HTTPS, meaning communication is encrypted using TLS.

Using Developer Tools → Network tab, we can see:

  • HTTPS request URL
  • Status code (304 Not Modified)
  • Response headers

The website loads over HTTPS, meaning all communication is encrypted using TLS.

In the browser’s Network tab, the main request returned 304 Not Modified, which means the browser already had a cached copy of the page and the server confirmed it hasn’t changed.

Security Feature

GitHub Pages automatically enforces HTTPS, which protects data integrity and confidentiality.

Summary

Our website is hosted publicly, DNS resolves the domain to an IP address, and communication occurs over HTTPS using TCP/IP.
