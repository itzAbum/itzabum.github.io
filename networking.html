How Networking Makes Our Website Work

1. DNS (Domain Name System)

This website is hosted using GitHub Pages under the domain: https://itzabum.github.io

When a user enters this domain in a browser, DNS resolves the domain name to an IP address. The browser then connects to that IP address using TCP and retrieves the website using HTTPS.

DNS Evidence

nslookup itzabum.github.io

The command above resolves the domain name to an IPv4 address, confirming that DNS is functioning correctly.

2. IP Addressing

The domain resolves to an IPv4 address provided by GitHub's hosting infrastructure.

IPv4 uses a 32-bit addressing format and enables communication between the client and server over the Internet.

This confirms that the website is publicly hosted and reachable.

3. Protocols (HTTP vs HTTPS)

This website uses HTTPS rather than HTTP. HTTPS encrypts communication between the browser and server using TLS.

Status Code

When inspected using browser Developer Tools, the site returns: 200 OK

Response Headers

HTTP/2 200 OK
Content-Type: text/html
Strict-Transport-Security: max-age=31536000

The 200 OK status confirms the request was successful. HTTPS ensures secure, encrypted communication over TCP/IP.

4. Security Implementation

GitHub Pages automatically enforces HTTPS. If a user attempts to access the site using HTTP, it redirects to HTTPS.

The Strict-Transport-Security header forces browsers to always use HTTPS for secure communication.
